Data breaks late.

Schemas drift silently.

Python ETL works fine until schema changes sneak in. Failures show up downstream, not at commit time.

Where we spend human time

  • Writing schemas
  • Updating dbt tests
  • Debugging mismatches
This is translation work: intent → schema → SQL checks. Repetitive, but requires reasoning.

GenAI is good at

reasoning — not guessing

Not prompt-to-schema magic. GenAI excels at understanding structure and change.

Schemas are

living artifacts

Humans already reason about schema evolution. SchemaForge automates that thinking.

What is SchemaForge?

  • Python-first
  • Schema reasoning engine
  • dbt rule generator
Input: real data. Output: executable dbt tests. Python and dbt stay central.

Where it fits

Python ETL → SchemaForge → dbt tests → Warehouse

Nothing is replaced. SchemaForge augments glue logic.
